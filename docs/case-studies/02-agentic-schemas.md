## 🎯 The Project

**agentic-schemas** is a proposed graph-structured architectural framework for building production-ready AI agent systems. It models 20 foundational agentic design patterns as an interconnected knowledge graph, with weighted edges representing pattern dependencies, composition strategies, and architectural flows.

Each pattern is defined through structured JSON schemas (vertices with metadata: layer, polarity, compute requirements, safety surface) and relationship mappings that reveal how patterns combine in practice — from basic prompt chaining and routing to complex multi-agent collaboration, RAG pipelines, guardrails, and reasoning techniques.

Includes:
- Interactive D3.js graph visualization
- Mermaid diagrams with hub/authority/bridge classifications
- Plain-English documentation covering trade-offs (cost, latency, failure modes)
- Composability rules and decision criteria for pattern selection

## 📦 What's Included

### 20 Agentic Design Patterns

**Core patterns (9):**
1. Prompt Chaining (assembly-line steps and validations)
2. Routing (smart triage to specialist agents)
3. Parallelization (split, normalize, merge)
4. Reflection (critic → revise → pass)
5. Tool Use (discover, authorize, execute, fallback)
6. Planning (milestones, dependencies, constraints)
7. Multi-Agent Collaboration (manager + roles + shared memory)
8. Memory Management (short, episodic, long-term; retrieval)
9. Learning and Adaptation (feedback → prompts/policies/tests)

**Reliability patterns (6):**
10. Goal Setting and Monitoring (KPIs, drift, course-correct)
11. Exception Handling and Recovery (classify, backoff, fallbacks)
12. Human-in-the-Loop (review cues and approvals)
13. Retrieval (RAG: parse, chunk, embed, rerank)
14. Inter-Agent Communication (protocols, IDs, expiry)
15. Resource-Aware Optimization (route by cost and complexity)

**Reasoning & meta patterns (5):**
16. Reasoning Techniques (CoT, ToT, self-consistency, debate)
17. Evaluation and Monitoring (golden sets, SLAs, drift)
18. Guardrails and Safety (PII, injection, sandboxing)
19. Prioritization (value × effort × urgency × risk)
20. Exploration and Discovery (map space, cluster, probe)

### Interactive Visualization
- D3.js knowledge graph
- Pattern dependency weights
- Hub/authority/bridge classifications
- Filter by layer, polarity, compute requirements

### Documentation
- Plain-English description per pattern
- When to use, when not to use
- Cost, latency, failure mode trade-offs
- Composition rules
